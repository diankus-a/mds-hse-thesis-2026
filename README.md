# mds-hse-thesis-2026
Репозиторий создан для работы с материалами магистерской ВКР

**Тема ВКР:** {{Сформулировать с научным руководителем}}
** Вариант темы от GPT**: «Разработка модели для автоматической идентификации дублирующихся событий и проверки стандартов именования в аналитических таксономиях»

**Идея проекта:** В работе разрабатывается инструмент, который анализирует созданную разметку событий, сравнивает её с существующей таксономией компании и выдаёт замечания по дублям и корректности названий.

**План работ:**
1. Определить тему (до 27.11):
   - Согласовать с научным руководителем
   - Загрузить в систему
2. Описать структуру ВКР (до 14.12):
   - Оглавление
   - Структура Введения
3.  Обзор литературы и существующих решений. Формат: отдельный документ (до 28.12):
   - Анализ методов поиска дублей (similarity, semantic matching).
   - Анализ методов проверки стандартизации / нормализации.
   - Анализ решений для data quality в продуктовой аналитике. Сформулировать ценностное предложение моего продукта.
   - Формирование выводов по обзору.
4. Формирование датасета (до 31.01):
   - Сгенерировать данные
   - Выложить
   - Провести EDA
   - Сформировать датасеты
5. Разработка baseline-моделей (до 22.02):
   - Определить baseline (TF-IDF similarity, простые правила и т.п.).
   - Выбрать метрики (accuracy, F1, cosine similarity threshold).
   - Провести эксперименты baseline → фиксировать результаты.
6. Разработка и оптимизация основной модели:
   - Создать ML-модель (например, BERT/Н-классификация/semantic similarity)
   - Провести обучение
   - Оптимизировать гиперпараметры
   - Сравнить с baseline.
   - Зафиксировать финальные метрики.
7. Черновая разработка программного решения (до 31.03):
   - Архитектура приложения
   - Структура классов, модули, API.
   - Прототип интерфейса.
   - Обоснование выбора технологий и архитектуры.
   - Код интерфейса и API.
   - Интеграция с моделью.
   - Тестирование.
8. Описать программное решение: 
   - Обоснование значимости предлагаемой системы
   - Подробное формализованное описание системы (общая архитектура, структура классов и их интерфейсы, объёмные характеристики и т.д.)
   - Описание системы с точки зрения пользователя.
   - Обоснование оптимальности выбранных решений, в том числе на основе экспериментального анализа сложности вычислений.
   - Соотнесение системы с известными аналогами по функциональности, эффективности и удобству использования
9. Финальная разработка программного решения
10. Объединение текста ВКР в один файл
11. Финальная версия текста (до 10.05)
12. Загрузка ВКР (до 20.05)
13. Подготовка презентации
